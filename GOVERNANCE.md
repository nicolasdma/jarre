# Governance

## Maintainer Model

Jarre is currently maintainer-led.

Maintainers are responsible for:

- triaging issues and pull requests
- enforcing code quality and project policies
- releasing changes and managing roadmap priorities
- enforcing the Code of Conduct

## Decision Process

- Small technical decisions are resolved in pull requests.
- Larger changes should start with an issue/discussion describing tradeoffs.
- Maintainers make final decisions when consensus is not reached.

## Contribution Access

- External contributions are accepted through pull requests.
- Direct push access is limited to maintainers.

## Project Direction

Roadmap priorities are guided by:

1. learner impact and product quality
2. reliability and security
3. contributor experience and maintainability
